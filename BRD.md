## 2. 사업 배경 및 추진 필요성

### 2.1 배경
- 광양 제2 LNG터미널 증설공사는 **#7·#8 LNG 저장탱크, LNG 부두, 154kV 수전설비**를 포함하는 대규모 복합 공사로, 다수 공구·공종이 동시 진행됨
- 기존 보고 체계는 **월간 PPT 보고서**(공구별 별도 작성)에 의존 → 임원·관리자가 핵심 지표를 빠르게 비교·추적하기 어려움
- 공정률·인력·안전·마일스톤이 각기 다른 자료에 흩어져 있어 **종합 현황 파악에 시간 소요**
- 38개월의 누적 데이터를 한 번에 비교할 수 있는 **장기 추세 뷰** 부재

### 2.2 추진 필요성
1. **단일 진실 원천(Single Source of Truth)** — 공구별 지표를 한 화면에서 비교
2. **장기 추세 가시화** — 38개월 시계열로 공정·인력·안전 패턴 모니터링
3. **즉시 보고 가능성** — 임원 보고 시 PPT 재가공 없이 화면 그대로 공유
4. **데이터 갱신 단순화** — 현장 담당자가 별도 도구 없이 브라우저에서 직접 수정·내보내기
5. **현장 사진 통합 열람** — 분기별 PPT에 흩어진 작업사진 341장을 월/공종별 필터로 접근

## 6. 사업 범위

### 6.1 In Scope
- 단일 정적 웹 대시보드 (`index.html` + `data.js` + `photos.js`)
- PPT → 사진/슬라이드 데이터 추출 Python 스크립트
- Cloudflare Pages 호스팅 + Cloudflare Access 접근 제어
- 보고 대상 기간: **2023-02 ~ 2026-03** (38개월), 이후 월간 갱신
- 현장 사진 통합 갤러리 (341장)

### 6.2 Out of Scope
- 사내 인증(SSO) 연동 — 외부 Cloudflare Access 사용
- 데이터 자동 수집 파이프라인(ERP·현장 시스템 직결)
- 다국어 지원
- 인쇄 전용 PDF 보고서 자동 생성
- 모바일 네이티브 앱

## 7. 가정 및 제약

### 7.1 가정
- 월별 PPT 보고서가 정해진 양식·일정에 맞춰 산출됨
- 데이터 입력자는 브라우저 + 텍스트 에디터 사용 가능
- Cloudflare Free Tier(50 사용자/월 500 빌드)로 충분
- 사내망에서 외부 CDN(jsDelivr / Google Fonts) 접근 가능

### 7.2 제약
- 사내망 보안 정책으로 인해 임의 외부 SaaS 도입 제한 → 정적 호스팅 + 이메일 OTP 방식 채택
- 빌드 도구·서버 운영 인력 부재 → **빌드리스(static)** 구조 필수
- OneDrive 동기화 환경에서 git 충돌 가능성 → 운영 가이드 별도 명시
- 일부 월 PPT는 DRM 보호로 자동 추출 불가 → 누락월(`data_gaps`) 명시적 처리
